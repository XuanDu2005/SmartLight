# SmartLight

SmartLight \u2014 Single Vendor E-Commerce Platform for lighting products in Vietnam.

This monorepo hosts the **bootstrap** of the SmartLight platform. **No business logic yet** \u2014 only the scaffolding required to start feature implementation.

> Source of truth: `../docs/05-software-architecture/` (highest), then `../docs/06-database-schema/`, then `../docs/04-api-design/`.

---

## Stack

| Layer | Technology |
|---|---|
| Language | TypeScript (strict) |
| Package manager | pnpm 9 (workspaces) |
| Monorepo | Turborepo |
| Backend | NestJS 10 (Node 20) |
| Frontend | React 18 + Vite |
| ORM | Prisma |
| Database | PostgreSQL 16 |
| Cache / Queue | Redis 7 (optional connection only) |
| Container | Docker (Compose for local dev) |

## Getting Started

### Prerequisites

- Node.js 20+
- pnpm 9+
- Docker & Docker Compose (recommended) **or** local PostgreSQL/Redis

### One-shot local dev (Docker)

```bash
cp .env.example .env
docker compose up -d --build
# Frontend:  http://localhost:5173
# Admin:     http://localhost:5174
# API:       http://localhost:4000
# Postgres:  localhost:5432
# Redis:     localhost:6379
```

### Local without Docker

```bash
cp .env.example .env
pnpm install
pnpm prisma:generate
pnpm dev
```

## Build, Test, Lint

```bash
pnpm build          # build all apps/packages via turbo
pnpm lint           # eslint
pnpm typecheck      # tsc --noEmit
pnpm test           # unit tests
pnpm format         # prettier --write
```

## What's Included at Bootstrap

- Workspace scaffolding (Turborepo + pnpm)
- TypeScript strict configuration
- ESLint + Prettier shared config
- NestJS API with **15 empty modules** (`identity`, `users`, `catalog`, `inventory`, `cart`, `checkout`, `order`, `payment`, `shipping`, `promotion`, `review`, `notification`, `media`, `admin`, `audit`) \u2014 no business logic
- Two React/Vite apps with routing + layout shells
- Prisma schema pointer
- Docker Compose for Postgres, Redis, API, Web, Admin
- `.env.example`

## What's NOT Included Yet

- Business logic (use cases, services, controllers logic)
- Auth flows
- Payment integrations
- AI features
- Migrations (to be generated by `prisma migrate dev`)
- E2E tests
