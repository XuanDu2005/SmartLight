## 1. Purpose

This document details the **Media API endpoints** for SmartLight. Covers file upload (single and batch), metadata retrieval, and signed direct upload for large files. Backed by Cloudinary.

---

## 2. Media Concepts

| Aspect | Implementation |
| --- | --- |
| Storage | Cloudinary (third-party) |
| Variants | Auto-generated (thumbnail, medium, large, original) |
| Max File Size | 10 MB for direct; 100 MB for signed |
| Allowed Types | `image/jpeg`, `image/png`, `image/webp`; `application/pdf` (V1.5+) |

---

## 3. Media Endpoints

### 3.1 EP-MED-001 — Single File Upload

| Field | Value |
| --- | --- |
| **Method** | POST |
| **URL** | `/v1/media/upload` |
| **Content-Type** | `multipart/form-data` |
| **Authentication** | Yes |
| **Idempotency** | Required |
| **Audit** | `media.uploaded` |
| **Related Entity** | media_file |

**Form Fields:**

| Field | Type | Required |
| --- | --- | --- |
| `file` | binary | Yes |
| `altText` | string | No (max 255) |
| `metadata` | JSON string | No |
| `folder` | string | No (logical grouping) |

**Response `201 Created`:**

```
{
  "data": {
    "id": "uuid",
    "url": "https://cdn.smartlight.vn/...",
    "secureUrl": "https://...",
    "filename": "product-001.jpg",
    "size": 102400,
    "mimeType": "image/jpeg",
    "width": 1600,
    "height": 1200,
    "variants": [
      { "name": "thumbnail", "url": "https://...", "width": 200, "height": 150 },
      { "name": "medium", "url": "https://...", "width": 800, "height": 600 },
      { "name": "large", "url": "https://...", "width": 1600, "height": 1200 },
      { "name": "original", "url": "https://...", "width": 1600, "height": 1200 }
    ],
    "altText": "...",
    "uploadedBy": { "type": "AdminUser", "id": "uuid" },
    "uploadedAt": "2026-07-03T15:30:00Z"
  }
}
```

**Errors:**
- `FILE_TOO_LARGE` (413)
- `INVALID_MIME_TYPE` (415)
- `UPLOAD_FAILED` (502)
- `QUOTA_EXCEEDED` (429)

**Business Rule:** BR-MED-001: Auto-optimization (resize + WebP fallback) by Cloudinary.

### 3.2 EP-MED-002 — Multiple File Upload

| Field | Value |
| --- | --- |
| **Method** | POST |
| **URL** | `/v1/media/upload-multiple` |
| **Authentication** | Yes |
| **Idempotency** | Required |

**Form Fields:**

| Field | Type | Required |
| --- | --- | --- |
| `files` | array of binary | Yes (max 10) |
| `altTexts` | array of string | No (parallel) |
| `metadata` | JSON string | No |

**Response `201 Created`:**

```
{
  "data": [
    { "id": "uuid1", ... },
    { "id": "uuid2", ... }
  ]
}
```

---

### 3.3 EP-MED-003 — Get Media Details

| Field | Value |
| --- | --- |
| **Method** | GET |
| **URL** | `/v1/media/{mediaId}` |
| **Authentication** | Optional |
| **Cache** | public, max-age=600 |

**Response `200 OK`:** Media metadata.

**Errors:** `MEDIA_NOT_FOUND` (404), `MEDIA_DELETED` (410)

---

### 3.4 EP-MED-004 — Delete Media

| Field | Value |
| --- | --- |
| **Method** | DELETE |
| **URL** | `/v1/media/{mediaId}` |
| **Authentication** | Yes (uploader or admin) |
| **Audit** | `media.deleted` |

**Response:** `204 No Content`

**Errors:**
- `MEDIA_IN_USE` (409) — referenced by product/category/etc.
- `FORBIDDEN` (403)

> Soft delete (Cloudinary asset deleted after 30 days for undo).

---

### 3.5 EP-MED-005 — Request Signed Upload URL

| Field | Value |
| --- | --- |
| **Method** | POST |
| **URL** | `/v1/media/{mediaId}/sign-upload` |
| **Authentication** | Yes |

For large files (browser direct upload to Cloudinary).

**Request Body:**

| Field | Type | Required |
| --- | --- | --- |
| `purpose` | enum | `large_upload`, `replace` |

**Response `200 OK`:**

```
{
  "data": {
    "mediaId": "uuid",
    "uploadUrl": "https://api.cloudinary.com/...",
    "signature": "...",
    "timestamp": 1720000000,
    "expiresAt": "..."
  }
}
```

> Client uploads directly; backend registers completion via webhook or polling.

---

### 3.6 EP-MED-006 — Get Media Variants

| Field | Value |
| --- | --- |
| **Method** | GET |
| **URL** | `/v1/media/{mediaId}/variants` |
| **Authentication** | None |
| **Cache** | public, max-age=1800 |

**Response `200 OK`:** List of available variants.

```
{
  "data": [
    { "name": "thumbnail", "url": "https://...", "width": 200, "height": 150, "size": 5120 },
    { "name": "medium", "url": "https://...", "width": 800, "height": 600, "size": 24576 },
    { "name": "large", "url": "https://...", "width": 1600, "height": 1200, "size": 102400 }
  ]
}
```

## 4. Image Variants Configuration

Standard variants generated by Cloudinary:

| Variant | Max Width | Format |
| --- | --- | --- |
| `thumbnail` | 200 | WebP fallback to original |
| `small` | 400 | Same |
| `medium` | 800 | Same |
| `large` | 1600 | Same |
| `original` | — | Original file |

Auto-applied by Cloudinary transformations. Variants generated async after upload.
