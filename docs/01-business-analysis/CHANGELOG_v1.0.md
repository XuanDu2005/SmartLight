## 1. Summary of Changes

This revision incorporates all Critical Required Changes from `REVIEW_REPORT.md`, generated by the Architecture Review Board following a comprehensive review of the Business Analysis documents (SRS.md, SYSTEM_FEATURES.md, BUSINESS_RULES.md, USER_STORIES.md, ACCEPTANCE_CRITERIA.md, GLOSSARY.md, ASSUMPTIONS.md, REQUIREMENTS_TRACEABILITY_MATRIX.md).

**Scope of revision:**
- 4 new modules defined (Inventory, Payments, Media, Tax)
- 32 new Functional Requirements added (FR-INV-001..009, FR-PAY-001..008, FR-MED-001..005, FR-TAX-001..005, Order State Machine)
- 17 new System Features added (SF-INV-001..005, SF-PAY-001..005, SF-MED-001..003, SF-TAX-001..004)
- 32 new Business Rules added (BR-INV-001..007, BR-PAY-006..011, BR-MED-001..003, BR-TAX-001..005, BR-GCH-001..004, BR-OSM-001..004, BR-MFA-001..003)
- 14 new User Stories added (US-GUEST-014..016, US-CUST-027..028, US-ADM-007..010, US-ORD-007..008, US-FIN-004..005, US-CAT-006)
- 17 new Acceptance Criteria added (AC-TAX-001..002, AC-INV-001..004, AC-PAY-001..002, AC-MFA-001, AC-MED-001, AC-ORD-001..002, AC-GCH-001..003; cookie reject referenced from existing AC-AC-071g)
- 25 new Glossary terms added
- 12 new Assumptions added
- Complete Order State Machine with Mermaid diagram and transition table
- Guest Checkout flow fully defined
- Admin MFA clarified (customer MFA explicitly excluded)
- MoSCoW prioritization applied across all features
- RTM updated with full traceability chains including Business Rules layer
- 22 features explicitly deferred from MVP to V1.1 / V1.5 / V2

Every traceability chain now includes the Business Rules layer:

```
AC → US → SF → BR → BG
```

Previously, the RTM traced from Feature directly to Business Goal. This was identified as a gap in the REVIEW_REPORT.md §11.

## 11. Breaking Changes

The following changes may require re-work during implementation:

| ID | Breaking Change | Impact | Mitigation |
| --- | --- | --- | --- |
| BC-01 | Inventory reservation moved from cross-cutting (SF-X-001) to dedicated module (SF-INV-002) | Implementation team must update data models; cross-cutting descriptor is removed | SF-INV-002 supersedes SF-X-001; ensure no duplicate reservation logic |
| BC-02 | Media features moved from Platform (SF-PLT-007) to dedicated Media module (SF-MED-001..003) | Cloudinary integration code must be refactored into media module | SF-MED-001..003 supersede SF-PLT-007; ensure CDN delivery is not disrupted |
| BC-03 | Customer MFA explicitly excluded from V1.0 (was ambiguous) | Auth team must NOT implement customer MFA; customer auth is password + email only | Document in tech spec; add feature flag `ENABLE_CUSTOMER_MFA` for V1.5 |
| BC-04 | 22 features deferred to V1.1+ | Product backlog must be re-prioritized; some planned work is now out of scope | Use feature flags for all deferred features; do not build dependencies on them |
| BC-05 | Order state machine enforces HTTP 409 on forbidden transitions | Backend must implement state machine validation; manual status override requires elevated role | Add state machine to order service; log all manual overrides |
